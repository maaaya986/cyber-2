Server (server.py)•Generates a random 30x30 maze using a recursive backtracker algorithm.•Manages player positions and IDs (starting from 10).•Handles concurrent connections using threads.•Broadcasts the updated map to all connected clients whenever a player moves.•Map values: 0 (Wall), 1 (Path), 2 (Food), 3 (Poison), 9 (Exit).Client (client.py)•Connects to the server via sockets.•Uses pygame to render the 30x30 map with 10x10 pixel squares.•Captures arrow key presses (UP, DOWN, LEFT, RIGHT) and sends them to the server.•Automatically updates the display when it receives a new map from the server.•Your player is highlighted in Yellow, while other players appear in Orange.How to run:1.Install pygame: pip install pygame2.Run the server first: python server.py3.Run one or more clients: python client.py

New Inheritance Architecture:1.UIElement (Base Class):◦Defines the core structure for any UI component (it has a rect and a color).◦Provides empty templates for draw(screen) and handle_event(event).2.Button (Inherits from UIElement):◦Adds specialized logic for rendering text in the center of the rectangle.◦Implements handle_event to specifically detect mouse clicks and return an action (like "UP" or "LOGIN").3.InputField (Inherits from UIElement):◦Adds logic for text input, labels, and password masking (*).◦Implements handle_event to manage focus (active state) and keyboard input.Why this is better:•Consistency: All UI elements now share the same interface (draw and handle_event).•Scalability: If you want to add a checkbox or a slider, you just inherit from UIElement.•Clean Loops: The login_loop and game_loop are much cleaner because they simply iterate through UI objects and call their shared methods.You can now run the server and then the client to see the refactored, object-oriented UI in action

Salt Column: Added a salt column to the users table in maze_game.db.2.Unique Salts: When a new user registers, the server generates a unique, cryptographically secure 16-byte random salt using secrets.token_hex(16).3.Hashed with Salt: The server now stores SHA256(password + salt) instead of just SHA256(password).4.Login Verification: During login, the server retrieves the user's unique salt, re-hashes the incoming password with it, and compares the result to the database.
